▶ 🔍 Why These BIOS Settings Matter Essential

Storage Controller (RAID → AHCI)

Modern Linux needs direct disk access. RAID mode hides the NVMe controller behind Intel's RAID abstraction, making the SSD invisible to Linux installers.

  • AHCI: Advanced Host Controller Interface - native SATA/NVMe protocol
  • RAID On: Intel RST (Rapid Storage Technology) - Windows-only feature
  • Impact: Without this change, fdisk -l shows no drives

Secure Boot Disable

Prevents unsigned bootloaders. Arch Linux uses unsigned GRUB by default.

  • Alternative: Set up custom Secure Boot keys (advanced)
  • Why disable: Simplifies installation, no compatibility issues

Absolute® (Computrace)

Intel vPro anti-theft technology with remote wipe capabilities.

  • Privacy concern: Phones home to Absolute Software servers
  • Permanently disabled: Cannot be re-enabled later
  • Alternative: Leave "Disabled" (can re-enable later)
▶ ⚙️ Step-by-Step BIOS Navigation Guide

🔹 Step 1: Accessing BIOS

  1. Power off laptop completely
    • Hold power button 5 seconds until fully off (not just sleep)
  2. Enter BIOS setup
    • Press power button once
    • Immediately start tapping F2 key rapidly (every 0.5 seconds)
    • Keep tapping until Dell BIOS blue screen appears
    • If Windows boots instead: Restart and try again, start F2 earlier

🔹 Step 2: Change Storage Mode (CRITICAL)

  1. Navigate to storage settings
    • You'll see tabs at top: General, System Configuration, Security, etc.
    • Click on "System Configuration" tab (or use arrow keys + Enter)
  2. Find SATA Operation setting
    • Scroll down to find "SATA Operation" (usually middle of list)
    • Current value will show "RAID On"
  3. Change to AHCI
    • Click on "RAID On" dropdown
    • Select "AHCI" from dropdown menu
    • Setting now shows: SATA Operation = AHCI ✓
  4. Apply the change
    • Click "Apply" button at bottom
    • Confirm popup: Click "Yes"

🔹 Step 3: Disable Secure Boot

  1. Navigate to Security tab
    • Click on "Security" tab at top
  2. Find Secure Boot option
    • Look for "Secure Boot" section
    • Find "Secure Boot Enable" setting
    • Current value will be "Enabled"
  3. Disable Secure Boot
    • Click on "Enabled" dropdown
    • Select "Disabled"
    • Setting now shows: Secure Boot Enable = Disabled ✓

🔹 Step 4: Disable Absolute/Computrace (Optional)

  1. Still in Security tab
    • Scroll down to find "Absolute®" section
  2. Check current setting
    • Look at current value (probably "Enabled")
    • ⚠️ Warning: "Permanently Disabled" cannot be undone!
  3. Choose your option
    • Safe choice: Select "Disabled" (can re-enable later)
    • Privacy choice: Select "Permanently Disabled" (irreversible)

🔹 Step 5: Save & Exit

  1. Save changes
    • Press F10 key (Save & Exit shortcut)
    • OR click "Exit" tab, then "Save & Exit"
  2. Confirm save
    • Popup asks: "Save configuration changes and exit?"
    • Click "Yes"
  3. System reboots
    • Laptop restarts automatically
    • Changes are now permanent
    • Windows may not boot (this is expected - you're installing Arch!)

Can't Access BIOS

  • Wrong timing: Start pressing F2 before Dell logo appears
  • USB keyboard: Try built-in laptop keyboard instead

▶ 💿 Creating the Arch USB Drive Prerequisite

🔹 Step 1: Download Arch Linux ISO

  1. Go to official download page
    • Visit: archlinux.org/download/
    • Look for section: "Download Release Information"
  2. Download the ISO file
    • File name: archlinux-YYYY.MM.DD-x86_64.iso
    • Size: ~650-700MB (changes monthly)
    • Choose a nearby mirror for faster download
  3. Verify download (recommended)
    • Download sha256sums.txt from same page
    • Windows: Use HashCheck or certutil -hashfile filename SHA256
    • Linux/Mac: sha256sum archlinux*.iso
    • Compare output with sha256sums.txt

🔹 Step 2: Prepare USB Drive

  1. Get a USB drive
    • Minimum: 2GB capacity
    • Recommended: USB 3.0 for faster boot
    • ⚠️ Will be completely erased!
  2. Insert USB drive
    • Plug into your computer
    • Note the drive letter (Windows) or device name (Linux/Mac)
    • Backup any important files first

🔹 Step 3A: Flash with Rufus (Windows - Recommended)

  1. Download and run Rufus
    • Get from: rufus.ie
    • No installation needed - just run the .exe
  2. Configure Rufus settings
    • Device: Select your USB drive
    • Boot selection: Click "SELECT" → choose your Arch ISO
    • Partition scheme: GPT
    • Target system: UEFI (non CSM)
    • File system: FAT32 (default)
    • Cluster size: Default
  3. Start the flash process
    • Click "START" button
    • If prompted about ISO Image mode: Select "Write in ISO Image mode"
    • Confirm warning about erasing USB
    • Wait 2-5 minutes for completion

🔹 Step 3B: Flash with dd Command (Linux/Mac)

  1. Find your USB device
    • Run: lsblk (Linux) or diskutil list (Mac)
    • Look for your USB drive (match the size)
    • Note device name (e.g., /dev/sdb or /dev/disk2)
    • ⚠️ Double-check device name to avoid data loss!
  2. Unmount the drive
    • Linux: sudo umount /dev/sdX* (replace X)
    • Mac: sudo diskutil unmountDisk /dev/diskN (replace N)
  3. Flash the ISO
    • Linux: sudo dd if=archlinux.iso of=/dev/sdX bs=4M status=progress
    • Mac: sudo dd if=archlinux.iso of=/dev/rdiskN bs=1m
    • Wait for completion (5-15 minutes)
    • Run: sudo sync to ensure data is written

🔹 Step 4: Verify USB Creation

  1. Safely eject USB
    • Windows: Right-click drive → "Eject"
    • Linux/Mac: sudo eject /dev/sdX
  2. Test the USB
    • Re-insert the USB drive
    • It should show as "ARCHISO" or similar
    • You should see files like vmlinuz-linux
  3. Ready for Dell 7420
    • Your bootable USB is ready
    • Label it "Arch Linux Live USB"
    • Keep it safe for the installation
▶ 🔧 Boot Priority & UEFI vs BIOS Important

Understanding Boot Modes

UEFI (Unified Extensible Firmware Interface):

  • Modern replacement for BIOS
  • Supports drives >2TB
  • Faster boot times
  • Better security features
  • Required for modern Arch installation

Legacy BIOS:

  • Older standard (1980s)
  • MBR partition table only
  • Limited to 4 primary partitions
  • Not recommended for new installations

Dell 7420 Boot Menu

F12 Boot Menu Options:
├── UEFI: USB Device (Arch Linux)  ← Choose this
├── USB Device (Arch Linux)       ← Legacy BIOS (avoid)
├── Windows Boot Manager
└── UEFI: Built-in EFI Shell

Verification Commands

  • cat /sys/firmware/efi/fw_platform_size → Should show "64"
  • efivar -l → Should list EFI variables
  • ls /sys/firmware/efi → Should show EFI directory
▶ 🚨 Common Boot Problems & Solutions Troubleshooting

USB Not Detected

  • Wrong USB port: Try different ports (USB 2.0 vs 3.0)
  • Fast Boot enabled: Disable in BIOS
  • USB not bootable: Re-flash with proper tool
  • Secure Boot blocking: Already disabled in BIOS setup

Black Screen After Boot Selection

  • Graphics driver issue: Add nomodeset to boot parameters
  • Memory problem: Try memtest86+ option first
  • Hardware incompatibility: Use older Arch ISO

Kernel Panic on Boot

  • Wait 30 seconds - sometimes resolves automatically
  • Try different USB port
  • Re-download and re-flash ISO
  • Check RAM with memtest86+

Wrong Boot Mode (Legacy instead of UEFI)

  • Symptoms: No /sys/firmware/efi directory
  • Fix: Reboot, press F12, select UEFI option
  • Prevention: Disable CSM (Compatibility Support Module) in BIOS

Emergency Recovery

If completely stuck:

  1. Hold power button 10 seconds (force shutdown)
  2. Remove USB drive
  3. Boot into BIOS (F2)
  4. Reset to defaults (F9)
  5. Reconfigure essential settings
  6. Try again with fresh USB

▶ 🚨 Wi-Fi Troubleshooting Guide Fix Issues

Problem: "No device found" or wlan0 missing

Diagnosis:

# Check if Wi-Fi hardware is detected
lspci | grep -i wireless
# Check if driver is loaded
lsmod | grep iwl
# Check network interfaces
ip link show

Solutions:

  1. Reload Wi-Fi driver: 
    rmmod iwlwifi
modprobe iwlwifi
  2. Enable interface manually: 
    ip link set wlan0 up
  3. Check firmware loading: 
    dmesg | grep iwlwifi

Problem: Can see networks but can't connect

Common causes & fixes:

  • Wrong password: Try re-typing carefully
  • Special characters: Use quotes: "My Network!"
  • Network congestion: Try 5GHz band if available
  • Router MAC filtering: Add laptop's MAC to router whitelist

Problem: Connection drops or unstable

Solutions:

  1. Check signal strength: 
    iwctl station wlan0 show
  2. Move closer to router during installation
  3. Disable power saving: 
    iw dev wlan0 set power_save off

Problem: "Operation not permitted" errors

Usually means interface conflicts:

# Kill conflicting processes
killall wpa_supplicant NetworkManager
# Restart iwd
systemctl restart iwd
# Try iwctl again

Problem: Hidden network not visible

Connect to hidden SSID manually:

# In iwctl
station wlan0 connect "HIDDEN_NETWORK_NAME"

Emergency: Complete Wi-Fi reset

If nothing works:

# Stop all network services
systemctl stop iwd systemd-networkd
# Remove saved networks
rm -rf /var/lib/iwd/*
# Restart
systemctl start iwd
# Try iwctl again from scratch

Choose wipe method
DECISION
▼
🤔 How thoroughly do you want to wipe?
A
Secure cryptographic wipe (recommended)
✅ Best for privacy. Writes random data. Takes 20-60 min. Cannot recover old data.
B
Quick wipe (just zap partition table)
⚡ Takes seconds. Old data technically recoverable. Fine if privacy not a concern.
Execute disk wipe
POINT OF NO RETURN
▼
🔀 Wipe method:
🔐 Secure (recommended)
⚡ Quick
⚠️ This takes 20-60 minutes. Do not interrupt!
Step 1 — open encrypted container over SSD:
root@archiso ~ # cryptsetup open --type plain -d /dev/urandom /dev/nvme0n1 to_be_wiped
Step 2 — fill with zeros (through the encrypted container = writes random data to disk):
root@archiso ~ # dd if=/dev/zero of=/dev/mapper/to_be_wiped bs=1M status=progress
Step 3 — close the container when done:
root@archiso ~ # cryptsetup close to_be_wiped
✅ dd will show progress. When it says "No space left on device" — it's done. Normal!
Just zap the partition table:
root@archiso ~ # wipefs -a /dev/nvme0n1
root@archiso ~ # sgdisk -Z /dev/nvme0n1
